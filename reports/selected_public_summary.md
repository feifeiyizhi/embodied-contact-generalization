# Selected Public Summary

The project has built a simulation evidence loop for contact-quality learning. The current public contribution is not a solved controller, but a disciplined way to define, measure, and diagnose soft-touch behavior.

Key public-facing contributions:

- a strict distinction between loose contact and valid stable soft touch;
- an episode-level taxonomy for hard contact, raw-contact-only behavior, unstable touch, and stable soft touch;
- a roadmap from contact-quality metrics to a full token control loop;
- a runnable diagnostic demo that turns sample metrics into a concise evidence report.

Current claim boundary:

- soft-touch success is not claimed;
- real-robot deployment is not claimed;
- direct Emu3.5 motor control is not claimed;
- DreamerV3 + Emu3.5 closed-loop execution is not claimed.

The repository is intended as Qingmiao support material: it shows how the project is being made measurable and technically inspectable.
