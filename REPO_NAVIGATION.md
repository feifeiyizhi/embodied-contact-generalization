# Repository Navigation

This repository is a compact public showcase, not a full experiment archive. It is meant to help a reviewer understand the research problem, the current technical route, and one runnable evidence-loop example.

Recommended reading order:

1. `README.md` for the overall claim boundary and repository scope.
2. `docs/project_overview.md` for the research question and Qingmiao-facing motivation.
3. `docs/architecture.md` for the full token control loop design.
4. `docs/contact_quality_taxonomy.md` for metric discipline and failure categories.
5. `soft_touch_evidence_loop_demo/README.md` for the runnable diagnostic demo.
6. `docs/roadmap.md` for next milestones.
7. `data_samples/DATA_AVAILABILITY.md` for public/private data boundaries.

The repository is intentionally small. Its value is in a defensible research spine:

- a contact-sensitive minimum task;
- strict evidence definitions;
- a taxonomy of failure modes;
- a concrete direction toward world-model-conditioned token action prediction;
- a small script that makes the evidence logic inspectable.

The full internal archive remains outside this repository because it contains raw traces, large artifacts, private paths, internal audit notes, and unpublished experimental details.
